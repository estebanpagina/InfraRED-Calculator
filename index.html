En Clase C, los primeros 3 octetos identifican la red y el último octeto identifica el equipo (host).

🌐

Dirección IP

Número único de 32 bits (4 octetos) que identifica a cada dispositivo en una red. Ejemplo: 192.168.1.25. Similar a la dirección postal de una casa. IPv4 · 32 bits · 4 octetos
🏘️

Subred

Una división lógica de una red IP más grande. Agrupa dispositivos cercanos para mejorar el rendimiento y la seguridad. Cada subred tiene su propia ID de red, rango de hosts y dirección de broadcast. Segmentación · Seguridad · Eficiencia
🎭

Máscara de Subred

Número de 32 bits que indica qué parte de la IP es la red y cuál es el host. En Clase C base es 255.255.255.0 (/24). Con subnetting cambia el último octeto: ej. 255.255.255.192 (/26). /24 → /25 → /26 → /27…
💻

Host

Cualquier dispositivo conectado a la red que tiene una dirección IP asignable: computadores, impresoras, teléfonos IP, cámaras, servidores. La fórmula de hosts útiles es 2^h − 2, restando la ID de red y el broadcast. Fórmula: 2^h − 2
📢

Broadcast

La última dirección de cada subred, reservada para enviar mensajes a todos los dispositivos de esa subred simultáneamente. Nunca se asigna a un host. En la subred 192.168.1.0/26 el broadcast es 192.168.1.63. No asignable · Último de la subred
🆔

ID de Red

La primera dirección de cada subred. Identifica a la subred en sí misma pero no se puede asignar a ningún equipo. En 192.168.1.0/26 el ID de red es 192.168.1.0. No asignable · Primera dirección
🏢

¿Para qué sirve el Subnetting?

Casos reales en empresas y redes TIC

🏫

Institución Educativa

Una escuela con 3 pisos necesita separar la red de administración, la de estudiantes y la de docentes. Con subnetting, cada área tiene su propia subred y no pueden acceder entre sí sin autorización.

Seguridad por segmento
🏥

Clínica u Hospital

Los equipos médicos críticos (monitores, ECG, rayos X) deben estar en una subred separada e independiente de la red WiFi de pacientes para garantizar disponibilidad y seguridad de los datos.

Disponibilidad crítica
🏭

Empresa Industrial

Una planta tiene subred de producción (PLCs, sensores), subred de oficinas y subred de cámaras IP. Cada una con máscara diferente según el número de equipos que la necesita.

Optimización de IPs
🏨

Hotel

Subred para huéspedes WiFi (muchos hosts), subred para gestión hotelera (PMS, cámaras, llaves digitales) y subred para restaurante/cocina con TPVs y tablets.

Segmentación por servicio
🏛️

Gobierno / Entidad Pública

Una alcaldía separa la red ciudadana (quioscos de acceso público) de la red administrativa interna donde se procesan datos sensibles de la comunidad.

Privacidad de datos
📡

ISP / Proveedor de Red

Un proveedor de internet asigna bloques de IPs de Clase C a sus clientes empresariales usando subnetting para aprovechar al máximo el espacio de direccionamiento disponible.

Gestión de IPs

1

Ejercicio: Calcular subredes a partir de hosts

Nivel: Básico · Clase C · Modo Equipos

📋 Enunciado: Se tiene la red 192.168.10.0 y se requieren 24 equipos por subred. Calcule la nueva máscara de subred, la cantidad de subredes creadas y construya la tabla de rangos para las primeras 4 subredes.
  1. Paso 1 — Sumar 2 al número de hosts:
    Los 2 extras corresponden a la dirección de Red y la de Broadcast, que no son asignables.
    24 + 2 = 26
  2. Paso 2 — Encontrar la potencia de 2 igual o superior a 26:
    2¹=2 · 2²=4 · 2³=8 · 2⁴=16 · 2⁵=32 ✅
    32 ≥ 26 → Los bits de host = 5
  3. Paso 3 — Calcular bits prestados (bits de subred):
    El cuarto octeto tiene 8 bits en total: 8 − 5 = 3 bits prestados
  4. Paso 4 — Nuevo prefijo CIDR:
    La Clase C base tiene prefijo /24. Le sumamos los bits prestados:
    /24 + 3 = /27
  5. Paso 5 — Calcular el octeto de la máscara (tabla de pesos):
    Encendemos los 3 bits más a la izquierda: 128 + 64 + 32 = 224
    Máscara final: 255.255.255.224
  6. Paso 6 — Calcular cuántas subredes se crean:
    2³ = 8 subredes creadas, con 2⁵ − 2 = 30 hosts útiles por subred.
  7. Paso 7 — Calcular el salto de red:
    2⁵ = 32 → Cada subred ocupa un bloque de 32 direcciones.

2

Ejercicio: Calcular hosts a partir de subredes requeridas

Nivel: Intermedio · Clase C · Modo Subredes

📋 Enunciado: La empresa "TIC Solutions" tiene la red 192.168.50.0 y necesita dividirla en 4 departamentos (Ventas, TI, RRHH, Gerencia). ¿Cuántos hosts útiles cabrán en cada subred?
  1. Paso 1 — Hallar los bits prestados para 4 subredes:
    2⁰=1 · 2¹=2 · 2²=4 ✅
    4 ≥ 4 → Los bits de red (prestados) = 2
  2. Paso 2 — Calcular el nuevo prefijo CIDR:
    /24 + 2 = /26
  3. Paso 3 — Calcular bits de host:
    8 − 2 = 6 bits de host
  4. Paso 4 — Calcular hosts útiles por subred:
    2⁶ − 2 = 64 − 2 = 62 hosts útiles
  5. Paso 5 — Calcular el octeto de la máscara:
    Encendemos los 2 bits más a la izquierda: 128 + 64 = 192
    Máscara final: 255.255.255.192
  6. Paso 6 — Salto de red:
    2⁶ = 64 → bloques de 64 direcciones.

Guía de la Tabla de Resultados

Qué significa cada columna y cada dato generado

# Número de Subred

Índice secuencial de cada subred generada. Comienza en 1. El total es igual a 2^(bits prestados).

ID Red

Primera dirección de la subred. Identifica a la subred en enrutadores. No se asigna a equipos.

Ej: 192.168.1.0 → es el nombre de la subred
Primer Útil

Primera dirección IP asignable a un equipo. Es ID de Red + 1.

Ej: 192.168.1.0 + 1 = 192.168.1.1
Último Útil

Última dirección IP asignable a un equipo. Es Broadcast − 1.

Ej: 192.168.1.63 − 1 = 192.168.1.62
Broadcast

Última dirección de la subred. Reservada para enviar mensajes a todos los hosts de esa subred. No se asigna a equipos.

Ej: 192.168.1.63 → broadcast de /26
Estado

"En Uso": Subred que satisface el requerimiento.
"Reserva Libre": Subred adicional creada por matemática binaria pero no solicitada.

Tutorial Oficial: ¿Cómo funciona InfraRED?

1. La Toma de Datos

Ingresas la IP Base (ej. 192.168.10.0). El sistema detecta que es Clase C y bloquea los primeros 3 octetos. Luego eliges si necesitas dividir por Equipos (Hosts) o por Subredes.

2. Procesamiento Matemático GFPI-F-135

Si buscas Equipos, el sistema suma +2 al requerimiento y busca la potencia de 2 que sea igual o superior (2^h >= Hosts + 2).

Si buscas Subredes, el sistema busca directamente la potencia de 2 igual o superior (2^n = Subredes) para hallar los bits prestados.

3. La Máscara (Binario a Decimal)

Usamos la tabla de pesos clásica del cuarto octeto: 128 | 64 | 32 | 16 | 8 | 4 | 2 | 1. La calculadora suma los valores de los bits prestados para calcular el nuevo prefijo y la máscara final.

4. Tabla de Saltos de Red

Calcula el tamaño del salto de red elevando 2 a los bits de host libres. Con este número construye la tabla clasificando automáticamente qué redes están en uso y cuáles quedan de reserva libre.
